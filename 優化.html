🚀 快速上手指南(5分鐘速覽)

📋 新手必讀 - 快速掌握核心概念

🎯 核心原則(記住這3點)

1️⃣ 往最早的排程靠齊

同一天、同一群組內,只對 最早的預約 發送簡訊

範例:

  • 客人當天有 10:30、14:00、16:00 三個預約
  • ✅ 只發送 10:30 的簡訊
  • ✗ 14:00 和 16:00 不發送

2️⃣ 同一群組只發一封

同一個門市群組內的預約,當天只發一封簡訊

範例:

  • 板橋醫美 09:00(群組3)
  • 板橋岩盤浴 16:00(群組3)
  • ✅ 只發送 09:00 的簡訊(同屬群組3)

3️⃣ 不同群組各發各的

不同群組之間互不影響,各自判斷最早預約

範例:

  • 板橋醫美 09:00(群組3)✅ 發送
  • 忠孝岩盤浴 14:00(群組1)✅ 也發送
  • 原因:不同群組,各發各的

🎨 操作要點(認識3種按鈕)

✅ 綠色按鈕 - 建議執行

系統建議您執行的操作(例如:確認發送)

⚠️ 紅色警告 - 不建議發送

客人在黑名單或有特殊情況,強烈建議不要發送

🔘 灰色按鈕 - 中性選項

取消操作或一般情況的選項

📌 常見情境(處理這3種狀況)

情境1:新增更早的預約

  • 系統行為: 會彈出確認視窗
  • 提示內容: 新預約將成為最早,原預約簡訊將取消
  • 您的操作: 確認是否要發送新的簡訊

情境2:取消最早的預約

  • 系統行為: 會自動補位
  • 提示內容: 次早預約將自動升級為最早
  • 您的操作: 確認是否要為次早預約發送簡訊

情境3:看到紅色警告

  • 系統行為: 顯示警告訊息
  • 提示內容: 客人拒收簡訊
  • 您的操作: 建議點擊「不發送」

🔐 簡訊發送前置條件 - 客戶資格檢查

⚠️ 通用規則 - 適用於所有簡訊發送場景

🎯 使用者快速指南(操作重點)

💡 看到警告就知道怎麼做

🚨 看到紅色警告 → 不要發送

  • 警告訊息: 此會員註記為不收簡訊的黑名單
  • 建議操作: 點擊紅色「不發送」按鈕
  • 原因: 客人明確拒絕接收簡訊

⚡ 看到黃色提示 → 注意檢查

  • 提示訊息: 排程衝突 或 自動補位通知
  • 建議操作: 確認資訊後,決定是否發送
  • 原因: 需要您確認簡訊內容是否正確

✅ 看到綠色按鈕 → 建議執行

  • 按鈕文字: 確認發送
  • 建議操作: 一般情況下可以直接發送
  • 原因: 系統判斷此客戶適合發送簡訊
🔧 詳細規則說明(工程師參考,點擊展開)

此檢查邏輯在任何簡訊發送前都會執行,包括:

  • ✓ 新建預約的簡訊提醒
  • ✓ 預約修改(改時間/改門市)的重新提醒
  • ✓ 取消最早預約後的次早補發
  • ✓ 新增更早預約的簡訊發送
  • ✓ 所有會進入「發出簡訊內容」介面的操作

客戶資格檢查規則

⚙️ 最終判斷流程 v2.5 (預設啟用)

系統會依序執行以下四關檢查，每一關卡都有不同的判斷邏輯和優先級。

🚫 第一關：黑名單檢查 (絕對優先)

檢查對象： 針對 Type: 標註 的預約

檢查欄位： Mark 欄位

黑名單關鍵字：

無 拒 沒 員工 醫師 離職 暫停 誤 簡訊 澳門 香港 大陸 空號 非本人 過世 往生 SPY 孕 客訴 歿 間諜

✋ 判斷結果： 若符合，直接引導取消發送，流程結束。

⚠️ 第二關：特殊規則檢查 (內含白名單例外)

檢查對象： 針對 Type: 標註 的預約

檢查欄位： Mark 欄位

觸發條件（符合任一即檢查）：

  • 包含「勿」但不包含「勿久等」
  • 包含「不」但不包含「推/call/摳/電/聯」

🌟 白名單例外： 若同時包含 天使 或 好友 關鍵字，則直接通過，流程結束。

✋ 判斷結果： 若觸發條件但無白名單，則引導取消發送，流程結束。

ℹ️ 第三關：電訪銷開關檢查

檢查條件： 僅當後台開關打開時執行

檢查欄位： promote 欄位

過濾值： 0, 10, 20

✋ 判斷結果： 若開關打開且符合條件，則引導取消發送，流程結束。

💡 注意： 此開關預設關閉，不檢查此項。

✅ 第四關：預設通過

若預約資訊能通過前面所有關卡的檢查。

✓ 判斷結果： 正常，可發送簡訊。

Type: 接受電訪/銷 - 前端開關控制 (預設關閉)

🎛️ 開關說明

  • 開關關閉(預設): 不檢查此項,所有 promote 狀態的客戶都可發送
  • 開關打開: 額外過濾 promote 為 0、10、20 的客戶
promote 值 狀態說明 開關關閉 開關打開
0 拒絕 ✓ 發送 ✗ 不發送
10 勿Call勿約 ✓ 發送 ✗ 不發送
20 勿call可line ✓ 發送 ✗ 不發送
其他 NULL 或其他值 ✓ 發送 ✓ 發送

⚠️ 實作重點提醒

  • 檢查時機: 在進入「發出簡訊內容」介面 之前 執行
  • 白名單優先: 天使/好友會員豁免所有黑名單規則
  • 關鍵字比對: 採用部分匹配(包含即符合)
  • 開關預設: 「接受電訪/銷」過濾預設關閉,避免誤攔
  • 不符合提示: 若客戶不符合資格,應顯示明確提示訊息
  • 記錄追蹤: 建議記錄因資格檢查而未發送的案例供後續分析

💡 與主流程的整合點

此檢查在以下時刻觸發:

  • 1️⃣ 新建預約後,準備發送提醒簡訊時
  • 2️⃣ 修改預約(時間/門市)後,準備重新發送時
  • 3️⃣ 取消最早預約,準備為次早預約補發時
  • 4️⃣ 新增更早預約,準備為新最早發送時

所有情況都會先通過此檢查,才進入「發出簡訊內容」確認介面

🎯 前端 UI 互動設計規範

📋 設計規範說明

根據預約新增、取消、修改等操作,前端需要判斷並顯示對應的提示訊息和按鈕樣式。以下定義標準互動場景的 UI 設計規範。

🎯 核心設計原則

1. 往最早的排程靠齊 - 以當天最早預約為準發送簡訊

2. 預約不會被擋 - 簡訊規則只影響發送邏輯,不阻止預約建立

3. 以「當天 + SYSGroup」為單位 - 同一客戶同一天同一群組只發送最早一筆

4. 不同 SYSGroup 彼此獨立 - 各群組的簡訊狀態互不影響

彈窗共用結構

📱 簡訊確認視窗架構

當預約操作會影響簡訊發送時,系統彈出統一結構的確認視窗:

  • 1. 標題列 - 簡訊發送 / 簡訊取消
  • 2. 提醒資訊區 - 顯示 0~N 個提醒框(依重要性排序)
  • 3. 簡訊預覽區 - 顯示/編輯簡訊內容與發送時間
  • 4. 按鈕列 - 左側取消 / 右側確認

顏色與狀態定義

狀態 用途 視覺呈現
danger 危險/高風險 違反客戶設定,可能帶來客訴 🚨 紅色系
warning 警告/提醒 排程衝突,自動補位通知 ⚡ 黃色/橘色系
success 正向引導 系統建議執行的操作 ✅ 綠色
default 中立/取消 保守不做變更 🔘 灰色
info 資訊提示 一般資訊說明 ℹ️ 藍色系
