技術實作重點 | 專注核心邏輯 | 快速上手

🎯 核心需求

目標：避免同一天多個預約重複發送簡訊，降低成本並改善顧客體驗
⚠️ 重要：每個預約對應一封簡訊的設計不變，僅透過「跳過發送」來達到優化

🔍 發送判斷邏輯（兩層規則）

規則一：消耗區域門市群組（優先判斷）

// 判斷邏輯 IF (預約門市 IN 門市群組) { checkGroupAppointments(同一天 AND 同一群組內的所有預約); IF (當前預約是最早的) { return "可發送"; } ELSE { return "跳過發送"; } }

規則二：單一門市規則（次要判斷）

// 判斷邏輯 IF (預約門市 NOT IN 任何門市群組) { checkStoreAppointments(同一天 AND 同一門市的所有預約); IF (當前預約是最早的) { return "可發送"; } ELSE { return "跳過發送"; } }

🛡️ 簡訊發送前置檢查流程

觸發時機：所有簡訊發送前都會執行（新建預約、修改預約、取消最早預約補發、新增更早預約）
預約時間優先級檢查
是否為當天在該範圍內的最早預約？
❌ 不是最早 → 跳過發送
✓ 是最早 → 繼續到第二關
客戶資格檢查
檢查客戶的 Type 和相關欄位設定
❌ 不符合資格 → 顯示提示並結束
✓ 通過檢查 → 繼續到第三關
用戶最終確認
進入「發出簡訊內容」介面，由用戶決定是否發送

🔐 客戶資格檢查邏輯

Type: 標註 - 黑白名單規則（預設啟用）

⭐ 白名單（最高優先級）
Mark 欄位包含關鍵字：天使 或 好友 → 強制通過，可發送簡訊
💡 即使同時包含黑名單關鍵字，白名單優先
🚫 黑名單（若無白名單才生效）
Mark 欄位包含以下任一關鍵字 → 不發送簡訊
// 基本黑名單關鍵字 const BLACKLIST = [ '無', '拒', '沒', '員工', '醫師', '離職', '暫停', '誤', '簡訊', '澳門', '香港', '大陸', '空號', '非本人', '過世', '往生', 'SPY', '孕', '客訴', '歿', '間諜' ];
⚙️ 特殊處理規則
關鍵字 規則 程式邏輯
勿 包含「勿」但不包含「勿久等」→ 不發送 mark.includes('勿') && !mark.includes('勿久等')
不 包含「不」但不包含「推/call/摳/電/聯」→ 不發送 mark.includes('不') && !['推','call','摳','電','聯'].some(k => mark.includes(k))
// 完整檢查邏輯範例 function checkCustomerEligibility(customer) { const mark = customer.mark || ''; // 1. 白名單優先 if (mark.includes('天使') || mark.includes('好友')) { return { eligible: true, reason: '白名單通過' }; } // 2. 檢查基本黑名單 for (const keyword of BLACKLIST) { if (mark.includes(keyword)) { return { eligible: false, reason: `包含黑名單關鍵字: ${keyword}` }; } } // 3. 特殊規則：勿 if (mark.includes('勿') && !mark.includes('勿久等')) { return { eligible: false, reason: '包含「勿」但非「勿久等」' }; } // 4. 特殊規則：不 if (mark.includes('不')) { const exceptions = ['推', 'call', '摳', '電', '聯']; const hasException = exceptions.some(k => mark.includes(k)); if (!hasException) { return { eligible: false, reason: '包含「不」但無例外關鍵字' }; } } return { eligible: true, reason: '通過所有檢查' }; }

Type: 接受電訪/銷（前端開關控制，預設關閉）

// 開關控制邏輯 const promoteFilterEnabled = false; // 預設關閉 function checkPromoteStatus(customer) { if (!promoteFilterEnabled) { return { eligible: true, reason: 'promote過濾已關閉' }; } // 開關打開時的檢查 const blockedValues = [0, 10, 20]; if (blockedValues.includes(customer.promote)) { return { eligible: false, reason: `promote值為 ${customer.promote}，拒絕狀態` }; } return { eligible: true, reason: 'promote狀態通過' }; }
promote值 開關關閉 開關打開
0（拒絕） ✓ 發送 ✗ 不發送
10（勿Call勿約） ✓ 發送 ✗ 不發送
20（勿call可line） ✓ 發送 ✗ 不發送
其他值或NULL ✓ 發送 ✓ 發送

// 簡訊發送前完整檢查流程 async function checkAndSendSMS(appointment, customer) { // ===== 第一關：預約時間優先級檢查 ===== const isEarliest = await checkIfEarliestAppointment(appointment); if (!isEarliest) { return { success: false, message: '此預約不是最早的，跳過發送' }; } // ===== 第二關：客戶資格檢查 ===== // 2.1 檢查標註（黑白名單） const markCheck = checkCustomerEligibility(customer); if (!markCheck.eligible) { return { success: false, message: `客戶資格不符: ${markCheck.reason}` }; } // 2.2 檢查promote狀態（若開關打開） const promoteCheck = checkPromoteStatus(customer); if (!promoteCheck.eligible) { return { success: false, message: `promote狀態不符: ${promoteCheck.reason}` }; } // ===== 第三關：顯示發送確認介面 ===== const userConfirmed = await showSMSConfirmDialog(appointment); if (!userConfirmed) { return { success: false, message: '用戶取消發送' }; } // 執行發送 return await sendSMS(appointment); } // 檢查是否為最早預約 async function checkIfEarliestAppointment(appointment) { const appointmentDate = appointment.date; const appointmentStore = appointment.store; // 查找門市群組 const storeGroup = findStoreGroup(appointmentStore); let sameRangeAppointments; if (storeGroup) { // 規則一：查找同一天、同一群組內的所有預約 sameRangeAppointments = await getAppointments({ date: appointmentDate, stores: storeGroup.stores }); } else { // 規則二：查找同一天、同一門市的所有預約 sameRangeAppointments = await getAppointments({ date: appointmentDate, store: appointmentStore }); } // 找出最早的預約 const earliestTime = Math.min(...sameRangeAppointments.map(a => a.time)); return appointment.time === earliestTime; }

⚡ 實作重點提醒

  • 檢查時機：在進入「發出簡訊內容」介面之前執行
  • 白名單優先：天使/好友會員豁免所有黑名單規則
  • 關鍵字比對：採用部分匹配（includes）
  • 開關預設：「接受電訪/銷」過濾預設關閉
  • 錯誤提示：不符合資格時顯示明確提示訊息
  • 日誌記錄：記錄因資格檢查而未發送的案例
  • 效能考量：群組查詢建議使用索引優化

📋 Quick Reference

  • 優先順序：白名單 > 黑名單 > 特殊規則 > 通過
  • 門市群組：5個群組，11個門市，優先判斷
  • 獨立門市：16個門市，採用同一門市規則
  • 檢查層級：3道關卡（優先級 → 資格 → 確認）
  • 開關控制：promote過濾預設關閉
