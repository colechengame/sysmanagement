# 簡訊管理系統 UI (SMS Management System UI)

> Dr. Shine 醫美診所 - 預約簡訊自動化管理系統

一個專為醫美診所設計的智能簡訊管理系統 UI,提供預約簡訊自動化、門市群組管理、成本優化等功能。

## 專案簡介

本系統旨在解決醫美診所在預約管理中的簡訊發送問題,透過智能排程和群組管理機制,有效降低簡訊成本並改善顧客體驗。

**核心理念:** 往最早的排程靠齊 - 同一天、同一群組內,只對最早的預約發送簡訊。

## 主要功能

### 🎯 核心功能
- **智能簡訊排程** - 自動判斷最早預約,避免重複發送
- **門市群組管理** - 支援27個門市,5個消耗區域群組
- **客戶資格檢查** - 黑名單/白名單管理,防止誤發送
- **自動補位機制** - 取消最早預約時,自動為次早預約發送簡訊
- **UI 互動設計** - 標準化的彈窗提示,清晰的視覺引導

### 💰 成本優化
- 目標節省 **35%** 簡訊成本
- 以月均 10,000 筆預約為例,可節省約 **2,450 元/月**
- 年度預估節省 **29,400 元**

## 系統規則

### 門市群組規劃

系統將 27 個門市劃分為 5 個消耗區域群組和 16 個獨立門市:

**消耗區域群組 (共 11 家門市)**
- **群組 1 - 忠孝區域:** 忠孝岩盤浴、忠孝光澤
- **群組 2 - 羅東區域:** 羅東岩盤浴、羅東光澤
- **群組 3 - 板橋區域:** 板橋醫美、板橋岩盤浴
- **群組 4 - 中壢彤顏區域:** 中壢彤顏健保、中壢彤顏醫美、中壢岩盤浴
- **群組 5 - 台中區域:** 台中光澤診所、台中岩盤浴

**獨立門市 (共 16 家)**
採用「同一門市規則」,包括南西光澤診所、永和彤顏診所、古亭光澤等

### 簡訊發送規則

#### 規則一:消耗區域群組規則(優先判斷)
- **同一天 + 同一門市群組**內的預約 → 只發送**最早**的一封簡訊
- 門市群組代表顧客可以在這些門市間「互相消耗療程」

#### 規則二:單一門市規則(次要判斷)
- 若門市**不在任何群組**內 → 採用「同一門市」規則
- **同一天 + 同一門市**的多個預約 → 只發送**最早**的簡訊

#### 重要前提
- 每個預約對應一封簡訊的設計不變
- 僅透過「跳過發送」來達到優化目的
- 預約不會被擋,簡訊規則只影響發送邏輯

## 快速開始

### 環境需求
- 現代瀏覽器 (Chrome, Firefox, Safari, Edge)
- 無需額外依賴,純靜態 HTML 文件

### 本地預覽

1. 克隆專案
```bash
git clone https://github.com/colechengame/sysmanagement.git
cd sysmanagement
```

2. 開啟主頁面
- 直接在瀏覽器中打開 `index.html`
- 或使用本地伺服器:
```bash
# Python 3
python -m http.server 8000

# Node.js (http-server)
npx http-server -p 8000
```

3. 瀏覽文檔
- 訪問 `http://localhost:8000` (如使用本地伺服器)
- 點擊側邊欄導航查看不同章節

## UI 互動設計

### 彈窗架構
系統使用統一的彈窗結構:
1. **標題列** - 簡訊發送/簡訊取消
2. **提醒資訊區** - 顯示 0~N 個提醒框(依重要性排序)
3. **簡訊預覽區** - 顯示/編輯簡訊內容與發送時間
4. **按鈕列** - 左側取消/右側確認

### 狀態定義
| 狀態 | 用途 | 視覺呈現 |
|---|---|---|
| **danger** | 違反客戶設定,可能帶來客訴 | 🚨 紅色系 |
| **warning** | 排程衝突,自動補位通知 | ⚡ 黃色/橘色系 |
| **success** | 系統建議執行的操作 | ✅ 綠色 |
| **default** | 中立/取消 | 🔘 灰色 |
| **info** | 一般資訊說明 | ℹ️ 藍色系 |

- **純前端實現** - 使用 HTML + CSS + JavaScript
- **響應式設計** - 支援桌面和移動設備
- **模組化架構** - 使用 iframe 載入子頁面
- **漸進式設計** - 側邊欄導航 + 內容區域分離
- **無框架依賴** - 輕量級,易於維護
