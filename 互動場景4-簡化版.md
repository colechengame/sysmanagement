### JavaScript

```javascript
// 顯示彈窗
function showSecondEarliestModal(earliestAppointment, newAppointment) {
    // 檢查最早預約是否沒有發簡訊
    if (!earliestAppointment.smsStatus ||
        earliestAppointment.smsStatus === 'cancelled' ||
        earliestAppointment.smsStatus === 'failed') {

        // 暫存資料
        window.currentEarliest = earliestAppointment;
        window.currentNew = newAppointment;

        // 顯示彈窗
        document.getElementById('secondEarliestModal').style.display = 'block';
    } else {
        // 直接新增預約（不發簡訊）
        createAppointment(newAppointment, { sendSMS: false });
    }
}

// 關閉彈窗
function closeSecondEarliestModal() {
    document.getElementById('secondEarliestModal').style.display = 'none';
}

// 處理「取消發送」
function handleNoSend() {
    // 直接新增次早預約，不發簡訊
    createAppointment(window.currentNew, { sendSMS: false });
    closeSecondEarliestModal();
    alert('已新增預約，未排程簡訊');
}

// 處理「確認發送」
function handleSend() {
    // 1. 為最早預約排程簡訊
    scheduleSMS(window.currentEarliest);

    // 2. 新增次早預約（不發簡訊）
    createAppointment(window.currentNew, { sendSMS: false });

    closeSecondEarliestModal();
    alert('已為最早預約排程簡訊，並新增次早預約');
}

// 排程簡訊
function scheduleSMS(appointment) {
    // 呼叫後端 API
    fetch('/api/appointments/' + appointment.id + '/schedule-sms', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            sendTime: calculateSendTime(appointment)
        })
    });
}

// 建立預約
function createAppointment(appointment, options) {
    // 呼叫後端 API
    fetch('/api/appointments', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            ...appointment,
            sendSMS: options.sendSMS || false
        })
    });
}
```

## 🔄 流程圖

```
使用者新增次早預約（16:00）
         ↓
系統檢查：最早預約（14:30）有沒有簡訊？
         ↓
    ┌────┴────┐
    ↓         ↓
  有簡訊    沒簡訊
    ↓         ↓
直接新增   顯示彈窗
(不發送)      ↓
         使用者選擇
              ↓
        ┌─────┴─────┐
        ↓           ↓
    取消發送    確認發送
        ↓           ↓
    新增預約    1. 最早預約排程簡訊
   (不發送)     2. 新增次早預約(不發送)
```

## 🎯 重點整理

### 1. 觸發條件
- 新增次早預約時
- 最早預約沒有簡訊（未發送/已取消/傳送失敗）

### 2. 提示內容
- 告知使用者：這是次早預約
- 說明操作：確認發送會為最早預約排程簡訊
- 顯示資訊：最早預約的時間、狀態、預計發送時間

### 3. 兩個選擇
- **取消發送**：都不發簡訊
- **確認發送**：為最早預約發簡訊

### 4. 設計原則
- ✅ 單一彈窗
- ✅ 清楚提示
- ✅ 兩個按鈕
- ✅ 簡單流程
