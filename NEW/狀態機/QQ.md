# 簡訊彈窗情境與文案維護表（PM 版 v1.1）

> 🎯 目標：  
> - PM 只需要維護「少數幾種邏輯情境的文字」。  
> - 同群組 / 跨群組會共用這些情境文案。  
> - 工程師用「文案代號 + 彈窗模式（guide-confirm / guide-cancel / info）」來實作。  

---

## 1. 總原則

1. **預約不會被擋**
   - 所有門市都可以直接幫客人建立預約。
   - 簡訊規則只影響「要不要發簡訊、發給哪一筆」，**不會阻止預約建立或保留**。

2. **簡訊以 SYSGroup 為單位計算**
   - 同一位客戶 + 同一個 SYSGroup，只會有一筆「最早預約」可發簡訊，其餘為 `none` 或異常狀態。

3. **不同 SYSGroup 彼此獨立**
   - 在 G2 做的任何操作（新增、取消），只會影響「G2 內部的簡訊狀態」，  
   - **不會改動 G0 / G1 / 其他群組的預約與簡訊排程**。

4. **跨群組是後端概念，前台只感覺在「當下門市」操作**
   - 使用者切到哪一個門市，就在那個門市底下操作預約與簡訊。
   - 文案只會講「這筆預約 / 目前的預約安排」，不會出現「群組 / SYSGroup」等技術用語。

5. **黑名單是共用規則**
   - 若勾選「黑名單」，所有彈窗：
     - 提示區塊最上方會多出一段黑名單警告文案。
     - 視覺樣式會改為「引導取消」（紅色樣式）。

6. **引導模式與彈窗模式對應**

| 引導模式 | 說明 | 預設彈窗模式（UI） |
|----------|------|---------------------|
| 引導確認發送 | 鼓勵這次發簡訊 | `guide-confirm`（綠色） |
| 引導取消發送 | 提醒這次不會發簡訊、偏向不要發 | `guide-cancel`（紅色） |
| 中立資訊 | 說明規則或現況，讓使用者自行判斷 | `info`（藍色） |

> ⚠️ 若「黑名單」勾選，UI 可能強制改成 `guide-cancel`，  
> 文案本身的「引導模式」仍如表中所示，供邏輯與測試參考。

## 5. 驗收心法（給 PM）

1. **看文案：有沒有講清楚在幹嘛？**
   - 不要出現「群組 / SYSGroup」這種技術字眼。
   - 以「更早預約」、「同時間預約」、「這次不會發簡訊」這種自然語句為主。

2. **看引導：顏色 + 文字方向一致**
   - 引導確認發送：綠色框＋文案是在「鼓勵這次發」。  
   - 引導取消發送：紅色框＋文案主軸是「這次不會發簡訊 / 建議不要發」。  
   - 中立資訊：藍色框＋說明規則或現況，讓使用者做選擇。

3. **看行為：邏輯是否符合規則**
   - 同一 SYSGroup 裡，任何操作後都應該只剩一筆「最早預約」是 `scheduled` / `delivered`。  
   - 其他預約簡訊狀態為 `none` 或異常（`failed` / `cancelled` 等）。  
   - 跨群組操作時，只影響目標群組，其他群組狀態不變。

4. **黑名單是否有「強烈勸退」感**
   - 黑名單勾選時，所有彈窗都應加上「黑名單警告前綴」。  
   - 視覺樣式偏向「引導取消發送」，取消按鈕要比確認按鈕更醒目。

> ✅ 只要以上四點都符合，整體簡訊彈窗行為即可視為已通過 PM 驗收。
