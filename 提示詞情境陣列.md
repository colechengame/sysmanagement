# 提示詞情境陣列 v1.0

> **版本**: v1.0
> **更新日期**: 2025-11-20
> **目的**: 系統化整理所有彈窗提示詞的完整內容，提供開發與測試時的標準參考。

---

## 1. 提示詞區塊樣式定義

| 模式 | 顏色主題 | 左邊框 | 背景色 | 圖示 | 文字顏色 | 使用時機 |
|------|---------|--------|--------|------|---------|---------|
| `guide-confirm` | 綠色 | `#388e3c` | `#e8f5e9` | ✓ | `#2e7d32` | 引導使用者確認發送 |
| `guide-cancel` | 紅色 | `#d32f2f` | `#ffebee` | ⚠️ | `#c62828` | 引導使用者取消發送<br>（含黑名單警告） |
| `info` | 藍色 | `#1976d2` | `#e3f2fd` | ℹ️ | `#1565c0` | 時間衝突提示<br>（資訊性提醒） |

## 2. 核心提示詞模板

### 2.1. 新增更早預約 (executeAddEarlier)

#### 情境 A：無時間衝突，正常新增更早預約

| 元素 | 內容 |
|------|------|
| **彈窗類型** | `guide-confirm` (綠色) |
| **彈窗標題** | `發送預覽` |
| **提示詞內容** | `⚠️ 將取代現有排程，原最早預約的簡訊將被取消。` |
| **問句** | `是否確認發送此預約的簡訊？` |
| **完整提示詞** | `✓ ⚠️ 將取代現有排程，原最早預約的簡訊將被取消。`<br><br>`是否確認發送此預約的簡訊？` |
| **確認後動作** | 舊最早→`cancelled`（若原為`scheduled`/`delivered`）<br>新預約→`scheduled` |
| **取消後動作** | 新預約→`none` |

#### 情境 B：時間衝突，新增時間與現有預約相同

| 元素 | 內容 |
|------|------|
| **彈窗類型** | `info` (藍色) |
| **彈窗標題** | `⚠️ 注意：時間衝突` |
| **提示詞內容** | `✓ 根據「先來後到」規則，此預約（ID 較小）為最早預約。新增的預約不會發送簡訊。` |
| **問句** | `是否要為此筆現有預約更新簡訊內容與發送時間？` |
| **完整提示詞** | `ℹ️ ✓ 根據「先來後到」規則，此預約（ID 較小）為最早預約。新增的預約不會發送簡訊。`<br><br>`是否要為此筆現有預約更新簡訊內容與發送時間？` |
| **確認後動作** | 舊預約（ID較小）→`scheduled`<br>新預約→`none` |
| **取消後動作** | 舊預約→保持原狀態<br>新預約→`none` |
| **特殊說明** | 顯示的簡訊預覽是**舊預約**（ID較小）的內容 |

### 2.4. 取消最早預約 (executeCancelEarliest)

| **彈窗標題** | `發送預覽` |

| **特殊說明** | 顯示的簡訊預覽是**補位預約**的內容 |

## 3. 黑名單疊加提示詞

### 3.1. 黑名單警告前綴

當 `isBlacklisted = true` 時，所有彈窗的提示詞前會疊加以下警告：

```
⚠️ 警告：此會員已被標記為黑名單，強烈建議取消發送！
```

### 3.2. 黑名單疊加規則表

| 原模式 | 黑名單後模式 | 提示詞變化 | 按鈕樣式變化 |
|--------|------------|-----------|-------------|
| `guide-confirm` | `guide-cancel` | 前綴加上黑名單警告<br>顏色變為紅色框 | 確認發送→白底黑字<br>取消發送→紅底白字+動畫 |
| `guide-cancel` | `guide-cancel` | 前綴加上黑名單警告<br>保持紅色框 | 保持原樣 |
| `info` | `guide-cancel` | 前綴加上黑名單警告<br>顏色從藍色變為紅色框 | 確認發送→白底黑字<br>取消發送→紅底白字+動畫 |

### 3.3. 黑名單提示詞範例

#### 原提示詞（無黑名單）
```
✓ 將取代現有排程，原最早預約的簡訊將被取消。

是否確認發送此預約的簡訊？
```

#### 疊加黑名單後
```
⚠️ 警告：此會員已被標記為黑名單，強烈建議取消發送！

✓ 將取代現有排程，原最早預約的簡訊將被取消。

是否確認發送此預約的簡訊？
```

## 4. 提示詞命名變數對照表

在程式碼實現中，提示詞使用以下變數名稱：

| 變數名稱 | 用途 | 範例值 |
|---------|------|--------|
| `promptText` | 主要提示內容 | `'⚠️ 將取代現有排程...'` |
| `question` | 確認問句 | `'是否確認發送此預約的簡訊？'` |
| `modalType` | 彈窗類型 | `'guide-confirm'` / `'guide-cancel'` / `'info'` |
| `body.title` | 彈窗標題 | `'發送預覽'` / `'⚠️ 注意：時間衝突'` |
| `body.text` | 額外說明文字（已廢棄） | `''`（已移除，整併至 promptText） |

## 5. 提示詞使用指南

### 5.1. 開發時

1. **新增操作時**：參考本文件對應情境的提示詞內容
2. **調用 `generateSmsPreview()`**：傳入 4 個參數
   ```javascript
   generateSmsPreview(
       apt,              // 預約物件
       promptText,       // 提示詞主要內容
       modalType,        // 彈窗類型
       question          // 確認問句
   )
   ```
3. **檢查黑名單**：確保 `showModal()` 正確處理 `isBlacklisted` 疊加

### 5.2. 測試時

1. **對照本文件**：確認實際顯示的提示詞與本文件一致
2. **測試黑名單疊加**：勾選黑名單後，所有彈窗應強制轉為 `guide-cancel` 模式
3. **驗證問句整併**：確認問句出現在提示詞區塊內，而非獨立於底部

### 5.3. 修改時

1. **更新本文件**：任何提示詞修改都應同步更新本文件
2. **保持一致性**：相同情境的提示詞用語應保持一致
3. **版本記錄**：重大修改時更新文件版本號
